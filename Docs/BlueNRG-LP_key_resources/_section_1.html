What is a BlueNRG-LP device?

  • The BlueNRG-LP is low power, Bluetooth Low Energy (LE) system on chip, with 32bit core ARM Cortex-M0+ processor @64MHz and it is compliant with Bluetooth Core Specification version 5.x.
  • It has 256 KB programmable Flash, 64 KB SRAM, MPU, and extensive peripherals set (ADC, AES, CRC, DMA,GPIO, I2C,IWDG, LPUART, PWR, RCC, RNG, RTC, SPI,TIM and USART).
  • It also supports the following Bluetooth Low Energy v5.x specification features: LE data length extension, LE 2 Mbps PHY, LE long range, LE extended advertising , GATT caching, Channel Selection Algorithm #2, Periodic Advertising and Periodic Advertising Sync Transfer, LE Ping procedure, LE L2CAP Connection-Oriented Channel, LE Power Control & Path Loss Monitoring.
  • User can refer to the BlueNRG-LP datasheet (DS13282) for detailed information about the device features, peripherals & Bluetooth LE capabilities.
  • The following BlueNRG-LP memory configurations are available:
    • BlueNRG-355xy (Flash: 256 Kb; RAM: 64 Kb)
    • BlueNRG-345xy (Flash: 256 Kb; RAM: 32 Kb)

What is a BlueNRG-LPS device?

  • The BlueNRG-LPS is low power, Bluetooth Low Energy (LE) system on chip, with 32bit core ARM Cortex-M0+ processor @64MHz and it is compliant with Bluetooth Core Specification version 5.x.
  • It has 192 KB programmable Flash, 24 KB SRAM, MPU, and extensive peripherals set (ADC, AES, CRC, DMA,GPIO, I2C,IWDG, LPUART, PWR, RCC, RNG, RTC, SPI,TIM and USART).
  • It supports the Bluetooth LE Direction Finding features (AoA/AoD) and other Bluetooth LE v5.x specification features: LE data length extension, LE 2 Mbps PHY, LE long range, LE extended advertising , GATT caching, Channel Selection Algorithm #2, Periodic Advertising and Periodic Advertising Sync Transfer, LE Ping procedure, LE Power Control & Path Loss Monitoring.
  • User can refer to the BlueNRG-LPS datasheet (DS13819) for detailed information about the device features, peripherals & Bluetooth LE capabilities.BlueNRG-332xy
  • The following BlueNRG-LPS memory configurations are available:
    • BlueNRG-332xy (Flash: 192 Kb; RAM: 24 Kb)

What is Bluetooth Low Energy (LE) technology?

  • For people new to the Bluetooth Low Energy technology, some fundamentals are available on Bluetooth LE Stack v3.x Library programming guidelines (PM0269), Bluetooth low energy technology section.
  • In particular, user can get a clear understanding about the Bluetooth LE features, layers (GAP, GATT, ATT, SM, L2CAP,HCI, Link & PHY) , devices roles (Master, Slave, Client, Server, Central, Peripheral) key information types (services & characteristics) and how to use them (read, write, notify, indicate).
  • For information about the BlueNRG-LPS, Bluetooth LE v3.1x Direction Finding features refer to the Bluetooth LE Stack v3.x Library programming guidelines (PM0269)

Basic evaluation steps for dummies (no specific Bluetooth LE know how is required)

Get the STSW-BNRGLP-DK, BlueNRG-LP/BlueNRG-LPS Development Kit (DK) HW/SW resources for evaluating BlueNRG-LP/BlueNRG-LPS Bluetooth LE capabilities:

  1. Order and buy an BlueNRG-LP/BlueNRG-LPS development kit.
  2. Get the STSW-BNRGLP-DK SW package and launch the related installer BlueNRG-LP_LPS DK-x.x.x.x-Setup.exe.

First basic step: Interact with Smarthpones world by running the ST Bluetooth LE Sensor application

  1. Power on the BlueNRG-LP/BlueNRG-LPS development kit by connecting to a PC USB port through a micro USB Cable or two batteries on battery holder (setting the JP2 jumpers accordingly).
  2. On selected smarthphone, install the ST BLE Sensor smarthphone app available on Google Play Store (Android) or Apple web site respectively:
    • ST BLE Sensor App for Android
    • ST BLE Sensor App for IOS
  3. Load and run the BlueNRG-LP/BlueNRG-LPS BLE_SensorDemo_BlueMS binary image on a BlueNRG-LP/BlueNRG-LPS kit, and look for the BlueNRG-LP/BlueNRG-LPS, BLE_SensorDemo_BlueMSapp application in discovery mode: BlueNRGLP device is detected on smarthphone display (prebuilt image is available on BlueNRG-LP/BlueNRG-LPS SDK, Firmware/BLE_Examples/BLE_SensorDemo_BlueMSapp/{STEVAL-IDB011V1|STEVAL-IDB012V1} folder)
  4. Tap on BlueNRGLP: BlueNRG-LP/BlueNRG-LPS ST Bluetooth LE Sensor application connects to the smarthphone and user can get graphical evidence of platform accelerometer (by plotting data) and environmental sensors (temperature and pressure plotting data).

Second basic step: use BlueNRG-LP/BlueNRG-LPS Navigator

  • BlueNRG-LP/BlueNRG-LPS Navigator is an friendly GUI which lets you directly download and run the selected prebuilt application binary image on the BlueNRG-LP/BlueNRG-LPS platforms without a SWD interface (it just requires a BlueNRG-LP/BlueNRG-LPS development kit connected to a PC USB port)
  • It provides demo applications description (html files) and access to board configurations and demo application source code if needed.
  • BlueNRG-LP/BlueNRG-LPS Navigator is available on STSW-BNRGLP-DK SW package on Start menu folder, ST BlueNRG-LP_LPS DK x.x.x, BlueNRG-LP/BlueNRG-LPS Navigator.
  • User is just requested to connect one or two BlueNRG-LP/BlueNRG-LPS development kit on PC USB ports and then select the specific demonstration application to be tested.

BlueNRG-LP/BlueNRG-LPS Power Save Modes

  • Three low power modes are provided to achieve the best compromise between low power consumption, short startup time and available wakeup sources:
    1. CPU-Halt mode (only the CPU is stopped. All device peripherals continue to operate and they can wake up the CPU when an interrupt/event occurs).
    2. Deepstop mode with timer: The device is in deep sleep and the timer clock sources (LSI or LSE) remain running. Wakeup is possible from GPIOs (PA0 to PA15 and PB0 to PB11), RTC, IWDG, Radio and the Hal Virtual Timers.
    3. Deepstop mode with no timer: The device is in deep sleep. All the peripherals and clock sources are turned off. Wakeup is possible only from GPIOs (PA0 to PA15 and PB0 to PB11).
  • BlueNRG-LP/BlueNRG-LPS, STSW-BNRGLP-DK SW package provides SW API (HAL_PWR_MNGR_Request()) in order to handle to power save modes according to the applications requirements and to the Bluetooth LE radio functionalities.
  • Refer to the BlueNRG-LP, BlueNRG-LPS Power Save Modes Application Note (AN5466) for more detailed information.
