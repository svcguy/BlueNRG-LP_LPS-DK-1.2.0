BlueNRG-LP/BlueNRG-LPS PC applications
Secure Bootloader tool

The Secure Bootloader tool is a PC application which uses the secure bootloader framework capabilities provided within the BlueNRG-LP/BlueNRG-LPS device UART bootloader, by allowing to generate the authentication keys, sign a binary image and activate the secure bootloader through OTP. Secure bootloader is a not reversible operation once it is activated on a BlueNRG-LP/BlueNRG-LPS device.

How to Run

  • User can run this PC application by clicking on the Secure Bootloader icon under: Start menu folder, ST BlueNRG-LP_LPS DK x.x.x, Secure Bootloader.
Figure 1: Secure Bootloader

Main user interface window

  • Secure Bootloader PC application allows to address the following features:
    1. Key generation
    2. Create Signed Image
    3. Store Key in OTP

Key generation

  • It allows to:
    1. Generate a new key pair (public and private)
    2. List all the existing key pairs on current database with option to delete or delete all
    3. Select the destination folder for the generated key pair
  • NOTEs:
    • Log tab is also available. It shows the outcomes of the selected operations

Create Signed Image

  • It allows to:
    1. Set the key pair (public and private) to create the file signature
    2. Select the file to be signed
    3. Set the name of the signed file
    4. Create the signed image
  • NOTEs:
    • Log tab is also available. It shows the outcomes of the selected operations

Store Key in OTP

  • It allows to:
    1. Select the device COM port
    2. Select the key pair (public and private) to be stored on device OTP
    3. Set the start address of the FW to be authenticated
    4. Specify if OTP must be locked or not
    5. Store the Key in OTP
  • NOTEs:
    1. Log tab is also available. It shows the outcomes of the selected operations.
    2. The secure boot feature is not reversible and cannot be disabled. When the secure boot feature is activated at reset, the device verifies whether the FW, in the main FLASH, is authenticated or not. The public key stored inside the OTP is used.
    3. The OTP cannot be changed, so, it is important to save the keys generated with the utility Key generation tab, because if a new FW update is required, the same keys must be used to generate the signature for the new signed FW, otherwise the secure bootloader detects the FW as not authenticated and it is not executed.
    4. Secure Bootloader utilities are available on Application/Secure_bootloader folder.
