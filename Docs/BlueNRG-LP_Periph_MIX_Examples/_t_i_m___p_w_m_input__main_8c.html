Detailed Description

Use of the TIM peripheral to measure an external signal frequency and duty cycle.

KEIL project

To use the project with KEIL uVision 5 for ARM, please follow the instructions below:

  1. Open the KEIL uVision 5 for ARM and select Project->Open Project menu.
  2. Open the KEIL project C:\Users\{username}\ST\BlueNRG-LP_LPS DK x.x.x\Projects\Periph_Examples\MIX\TIM\TIM_PWMInput\MDK-ARM\{STEVAL-IDB011V1|STEVAL-IDB012V1}\TIM_PWMInput.uvprojx
  3. Select desired configuration to build
  4. Select Project->Rebuild all target files. This will recompile and link the entire application
  5. To download the binary image, please connect an USB cable in your board (CMSIS-DAP upgrade).
  6. Select Project->Download to download the related binary image.
  7. Alternatively, open the BlueNRG-LP Flasher utility and download the built binary image.

Usage

Use of the TIM peripheral to measure an external signal frequency and duty cycle.

The TIM1CLK frequency is set to 64MHzk, the Prescaler is 0 so the counter clock is 64MHz.

TIM1 is configured in PWM Input Mode: the external signal is connected to TIMx Channel2 used as input pin. To measure the frequency and the duty cycle, we use the TIM1 CC2 interrupt request, so in the TIM1_IRQHandler routine, the frequency and the duty cycle of the external signal are computed.

"uwFrequency" variable contains the external signal frequency: TIM1 counter clock = SystemCoreClock/1, uwFrequency = TIMx counter clock / TIMx_CCR2 in Hz

"uwDutyCycle" variable contains the external signal duty cycle: uwDutyCycle = (TIMx_CCR1*100)/(TIMx_CCR2) in %.

The minimum frequency value to measure is (TIMx counter clock / CCR MAX) = (64 MHz/1)/ 65535 = 915,5 Hz

BlueNRG_LP-EVB Set-up

  • TIMx_CH2 the external signal must be connected to TIMx Channel2 used as input pin.

Connect USART1 TX/RX to respectively RX and TX pins of PC UART (could be done through a USB to UART adapter) :

  • Connect BlueNRG_LP board USART1 TX pin to PC COM port RX signal
  • Connect BlueNRG_LP board USART1 RX pin to PC COM port TX signal
  • Connect BlueNRG_LP board GND to PC COM port GND signal

Launch serial communication SW on PC (as HyperTerminal or TeraTerm) with proper configuration :

  • 115200 bauds
  • 8 bits data
  • 1 start bit
  • 1 stop bit
  • no parity
  • no HW flow control
