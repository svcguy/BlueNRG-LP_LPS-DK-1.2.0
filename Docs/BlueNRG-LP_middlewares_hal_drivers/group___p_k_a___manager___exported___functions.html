PKAMGR_StartP256PublicKeyGeneration.

Function used to request the generation of a new ECC Public Key. The random number (i.e. the Secret part of the Key) necessary is generated inside this procedure itself.

Parameters
funcCB- callback function that BLEPS Library makes available to BLEPLAT HAL, that is called when the PKA operation requested has been completed (either successfully or not).
Returns
Status of the PKA peripheral w.r.t. the requested operation
  • SUCCESS in case the PKA peripheral is available and ready to execute the operation.
  • ERROR_BUSY in case the PKA peripheral is BUSY with another operation that could have been requested by the BLEPS Library or by the End-User Application.
Note
Called by BLEPS Library function: hci_le_read_local_p256_publicKey< >.

typedef void(* PKAMGR_funcCB) (PKAMGR_ResultStatus errorCode, void *args)

PKA Callback pointer definition.

Parameters
errorCode- indicates the status of the PKA operation just completed.
args- reference to a copy of the output/result of the PKA operation. It may be either: a) an array of 3x32 octets long, containing the Local Public Key, in case of P256 Public Key generation, formatted as follows: | Secret Key | Public Key X-coord | Public Key Y-coord | | (32 octets) | (32 octets) | (32 octets) | b) an array of 32 octets long, containing the DHKey-check result, in case of DHKey check operation, formatted as follows: | DHKey check | | (32 octets) | | N.A. | DHKey check | N.A. | | (32 octets) | (32 octets) | (32 octets) | The PKA processing output/result is reported as a linear array of octets, without any indication of the data length. The caller of the PKA operations shall read/decode the result, including the data length, based on the last operation requested.
Note
1 - ERROR_PARAMETERS should not be reported to the callback, assuming that the input parameters (passed for DHKey computation) are checked when calling PKAMGR_StartP256DHkeyGeneration.
2 This model is based on a unique and generic callback function, that reports the PKA output/result without enforcing a specific function argument typing (preferred option). Otherwise, 2 specific function callbacks may be defined, each one with specific function arguments depending on the specific operation requested.Pointer to a PKA callback function
