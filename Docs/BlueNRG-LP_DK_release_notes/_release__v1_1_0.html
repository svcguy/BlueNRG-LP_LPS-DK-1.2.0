BlueNRG-LP/BlueNRG-LPS Development kits (STSW-BNRGLP-DK)
Release Notes 1_1_0/ 14-May-2021 (BlueNRG-BLE Stack Library v3.1)

New Features/Changes

  • This section lists the new features/changes compared to previous BlueNRG-LP DK release:
    • STSW-BNRGLP-DK, BlueNRG-LP SDK v1.1.0 release supporting BlueNRG-BLE Stack Library v3.1 (Bluetooth Low Energy stack).
    • Added support for WiSE-Studio IDE toolchain (gcc based) and all related WiSE-Studio demonstration applications projects
    • Added Secure Bootloader GUI providing the secure bootloader capabilities (key generation, file signing and secure bootloader activation)
    • Added Bluetooth Low Energy Clock Synchronization and LE Power Control demo applications and related documentation
    • Added BLE_STACK_CUSTOM_CONF option for allowing user to define a specific set of Bluetooth Low Energy modular configuration options.
    • Updated OTA Reset and Service Manager framework in order to support the authentication features provided by BlueNRG-LP Secure Bootloader.
    • Updated Peripheral Drivers files names for LL, HAL and MIX with prefix: rf_driver
    • Added PVD LL demonstration application
    • Updated LPUART and TIM peripheral drivers
    • Updated ST BLE Sensor Demo with BlueVoice Audio Library and BLE Beacon demo with PeriodicAdv configuration
    • Updated BlueNRG-X Radio Init Wizard for aligning to BLE stack v3.1
    • Updated OTA_btl.c file and OTA Service Manager size to 94 Kbytes due to BLE stack v3.1 size increase and also for covering WiSE-Studio gcc toolchain
    • Updated Bluetooth LE Examples, ModulesInit() function for properly initializing BLE controller, AES, PKA and RNG drivers
    • Updated miscutil.c with new APIs for getting RSSI value
    • Added Bluetooth LE stack v3.0 to v3.1 general migration guideline for user application.
  • Update BlueNRG-LP datasheet (Rev 4) and BlueNRG-LP device Reference Manual (RM0479 Rev2)
  • Updated Bluetooth Low Energy v3.x Programming Manual (PM0269 Rev2)
  • Updated The BlueNRG-LP UART bootloader protocol Application Note (AN5471 Rev3)
  • Updated BlueNRG-LP Kits User Manual (UM2375 Rev5)
  • Updated Bringing up the BlueNRG-LP device Application Note (AN5503 Rev2)
  • Updated USB_CMSISDAP FW version to 1.8

  • Updated linker files for supporting BlueNRG-345 device (BlueNRG-LP 32K RAM): refer to BlueNRG-LP Key Resources, 4) Design & Bringing Up custom BlueNRG-LP PCB section, for how to migrate to BlueNRG-345.

  • NOTES:
    • Start menu folder for STSW-BNRGLP-DK v1.1.0, SW package is ST BlueNRG-LP DK 1.1.0
    • All SW package folders (Application, Docs, Drivers, Firmware, Middlewares, Project, Utility) are located, by default, under C:\Users\{username}\ST\BlueNRG-LP DK 1.1.0:
      • "Release folder" on STSW-BNRGLP-DK 1.1.0 "Start Menu" allows to get direct access to such folders.
    • Before using a BlueNRG-LP platform, please align the USB_CMSISDAP to the latest available version 1.8.
    • After upgrading the USB_CMSISDAP FW, please unplug and replug again the BlueNRG-LP, STEVAL platform on a PC USB port, and wait for complete proper board recognition with associated PC drivers.

  • BlueNRG-LP Radio Init Wizard supporting BlueNRG-LP devices and Bluetooth LE stacks v3.x:
    • It allows to properly configure the BLE_STACK_Init() API parameters in an user friendly way and to generate the related application *_config.h file to be used on user C application.
    • Please refer to the following documents for more information about the Bluetooth LE radio initialization parameters to be configured:
      • Bluetooth LE stack v3.x Stack Memory Manager HTML documentation
      • GATT database size guidelines HTML documentation

Secure Bootloader PC application

  • Secure Bootloader GUI PC application supporting BlueNRG-LP devices Secure bootloader framework :
    • It allows to generate the authentication keys, sign a binary image and activate the secure bootloader through OTP.

PC application/utilities (available on standalone SW packages)

  • BlueNRG GUI SW package (STSW-BNRGUI): it contains the BlueNRG GUI PC application and script launcher utility supporting also BlueNRG-LP device.
  • BlueNRG-X Flasher SW package (STSW-BNRGFLASHER): it contains PC utilities for device Flash read, erase, verify and programming operations.
  • BlueNRG current consumption estimation tool SW package (STSW-BNRG001): it provides a way to predict the current consumption of the device.

      4. From ST-LINK menu, select Firmware Upgrade and Device Connect. Click on Yes >>> tab for performing the ST-LINK firmware upgrade.

SDK Known problems and limitations

  • This section lists the SDK SW issues known at release time. Issues found after this release are listed as solved or limitations on newer releases:
    • ID 3624: High power consumption in RX state when high power mode is used (i.e. SMPS voltage level 1.9V)
    • ID 3556: RSSI measurements are not correct when High Power mode is used (i.e. SMPS voltage level 1.9V)
    • ID 3447: Free RTOS framework is not supported on WiSE-Studio context
    • ID 3079: Limitation with allocation of advertising buffers with GCC (WiSE) toolchain
    • ID 795: BlueNRG APP v 1.1 (smarthphone app) doesn't work on Android 10

    • ID 0216: if a BlueNRG-LP HW board is configured for using SMPS (an inductor is connected between the pin VLXSD and the pin VFBSD), but user application sets the SMPS OFF SW configuration, the device doesn't work and it is not possible to connect to it. Possible workaround:

      • Provide a temporary VBLUE to the pin VFBSD in order to start up the device providing external power to the LDO1V2 that power up the digital domain. After the startup it is possible to load the CORRECT SMPS setting.
