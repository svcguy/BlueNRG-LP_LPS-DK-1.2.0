Detailed Description

How to periodically update the IWDG reload counter and simulate a software fault that generates an MCU IWDG reset after a preset laps of time.

KEIL project

To use the project with KEIL uVision 5 for ARM, please follow the instructions below:

  1. Open the KEIL uVision 5 for ARM and select Project->Open Project menu.
  2. Open the KEIL project C:\Users\{username}\ST\BlueNRG-LP_LPS DK x.x.x\Projects\Periph_Examples\HAL\IWDG\IWDG_WindowMode\MDK-ARM\{STEVAL-IDB011V1|STEVAL-IDB012V1}\IWDG_WindowMode.uvprojx
  3. Select desired configuration to build
  4. Select Project->Rebuild all target files. This will recompile and link the entire application
  5. To download the binary image, please connect an USB cable in your board (CMSIS-DAP upgrade).
  6. Select Project->Download to download the related binary image.
  7. Alternatively, open the BlueNRG-LP Flasher utility and download the built binary image.

Usage

How to periodically update the IWDG reload counter and simulate a software fault that generates an MCU IWDG reset after a preset laps of time.

At the beginning of the main program the HAL_Init() function is called to reset all the peripherals, initialize the Flash interface and the systick.

The IWDG time-out is set to 762 ms (the time-out may varies due to LSI frequency dispersion).

The Window option is enabled with a window register value set to 400 ms. To prevent a reset, the down-counter must be reloaded when its value is: -lower than the window register value (400ms) -greater than 0x0 The IWDG counter is therefore refreshed each 450 ms in the main program infinite loop to prevent a IWDG reset (762 - 450 = 312 within the interval). LED2 is also toggled each 450 ms indicating that the program is running. LED3 will turn on if any error occurs.

An EXTI Line is connected to a GPIO pin, configured to generate an interrupt when the User push-button (PUSH1) is pressed.

Once the EXTI Line event occurs by pressing the User push-button (PUSH1), the refresh period is set to 200 ms. That will make refresh being outside window value. As a result, when the IWDG counter is reloaded, the IWDG reset occurs.

In the ISR, a write to invalid address generates a Hard fault exception containing an infinite loop and preventing to return to main program (the IWDG counter is not refreshed). As a result, when the IWDG counter falls to 0, the IWDG reset occurs. If the IWDG reset is generated, after the system resumes from reset, the message "IWDGRST flag is set." is showed over the UART.

If the EXTI Line event does not occur, the IWDG counter is indefinitely refreshed in the main program infinite loop, and there is no IWDG reset.

In order to make the program work, you must do the following:

  • Launch serial communication SW on PC
  • Flash the project in the Board
  • Press the RESET button

BlueNRG_LP-EVB Set-up Connect USART1 TX/RX to respectively RX and TX pins of PC UART (could be done through a USB to UART adapter) :

  • Connect BlueNRG_LP board USART1 TX pin to PC COM port RX signal
  • Connect BlueNRG_LP board USART1 RX pin to PC COM port TX signal
  • Connect BlueNRG_LP board GND to PC COM port GND signal

Launch serial communication SW on PC (as HyperTerminal or TeraTerm) with proper configuration :

  • 115200 bauds
  • 8 bits data
  • 1 start bit
  • 1 stop bit
  • no parity
  • no HW flow control
