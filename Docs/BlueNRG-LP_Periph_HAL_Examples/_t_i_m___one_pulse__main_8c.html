Use of the TIM peripheral to generate a single pulse when an external signal falling edge is received on the timer input pin.

KEIL project

  1. Open the KEIL uVision 5 for ARM and select Project->Open Project menu.
  2. Open the KEIL project C:\Users\{username}\ST\BlueNRG-LP_LPS DK x.x.x\Projects\Periph_Examples\HAL\TIM\TIM_OnePulse\MDK-ARM\{STEVAL-IDB011V1|STEVAL-IDB012V1}\TIM_OnePulse.uvprojx

  4. Select Project->Rebuild all target files. This will recompile and link the entire application

Usage

Use of the TIM peripheral to generate a single pulse when an external signal falling edge is received on the timer input pin.

The external signal is connected to TIMx_CH2, and a falling edge on this input is used to trigger the Timer. The One Pulse signal is output on TIMx_CH1.

The delay value is fixed to: Delay = CCR1/TIM1 counter clock = 16383 / 1000000 [sec] = 16 ms

The pulse value is fixed to : Pulse value = (TIM_Period - TIM_Pulse)/TIM1 counter clock = (65535 - 16383) / 1000000 [sec] = 49ms

Connecting the Trigger pin to TIMx TI2 allows to trigger TIMx counter by pressing the User push-button.

Connect the following pins to an oscilloscope to monitor the different waveforms:

  • Trigger pin
  • TIMx_CH1
  • TIMx_CH2
