Detailed Description

How to use a DMA channel to transfer a word data buffer from embedded SRAM to embedded SRAM. The peripheral initialization uses LL unitary service functions for optimization purposes (performance and size).

KEIL project

To use the project with KEIL uVision 5 for ARM, please follow the instructions below:

  1. Open the KEIL uVision 5 for ARM and select Project->Open Project menu.
  2. Open the KEIL project C:\Users\{username}\ST\BlueNRG-LP_LPS DK x.x.x\Projects\Periph_Examples\LL\DMA\DMA_MemToMem\MDK-ARM\{STEVAL-IDB011V1|STEVAL-IDB012V1}\DMA_MemToMem.uvprojx
  3. Select desired configuration to build
  4. Select Project->Rebuild all target files. This will recompile and link the entire application
  5. To download the binary image, please connect an USB cable in your board (CMSIS-DAP upgrade).
  6. Select Project->Download to download the related binary image.
  7. Alternatively, open the BlueNRG-LP Flasher utility and download the built binary image.

Usage

How to use a DMA channel to transfer a word data buffer from embedded SRAM to embedded SRAM. The peripheral initialization uses LL unitary service functions for optimization purposes (performance and size).

Then the LED_Init() function is used to initialize the LED2.

Then the Configure_DMA() function is used to configure the DMA1 to transfer the contents of a 32-word data buffer stored in embedded SRAM to the reception buffer declared in RAM.

The start of transfer is triggered by software(LL_DMA_EnableChannel()). DMA1 Channel1 memory-to-memory transfer is enabled. Source and destination addresses incrementing is also enabled. The transfer is started by setting the channel enable bit for DMA1_Channel1. At the end of the transfer a Transfer Complete interrupt is generated since it is enabled and the callback function (customized by user) is called.

Finally, aSRC_Const_Buffer and aDST_Buffer are compared through Buffercmp() in order to check buffers correctness.

BlueNRG_LP-EVB's LED2 can be used to monitor the transfer status:

  • LED2 is turn on if DMA data transferred consistency.
  • LED2 is blinking every 1 sec in case of error.

In order to make the program work, you must do the following:

  • Launch serial communication SW on PC
  • Flash the project in the Board
  • Press the RESET button

BlueNRG_LP-EVB Set-up Connect USART1 TX/RX to respectively RX and TX pins of PC UART (could be done through a USB to UART adapter) :

  • Connect BlueNRG_LP board USART1 TX pin to PC COM port RX signal
  • Connect BlueNRG_LP board USART1 RX pin to PC COM port TX signal
  • Connect BlueNRG_LP board GND to PC COM port GND signal

Launch serial communication SW on PC (as HyperTerminal or TeraTerm) with proper configuration :

  • 115200 bauds
  • 8 bits data
  • 1 start bit
  • 1 stop bit
  • no parity
  • no HW flow control
