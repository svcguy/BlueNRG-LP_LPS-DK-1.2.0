Detailed Description

Configuration of a timer to generate a fixed number of pulses in Output Compare mode.

KEIL project

To use the project with KEIL uVision 5 for ARM, please follow the instructions below:

  1. Open the KEIL uVision 5 for ARM and select Project->Open Project menu.
  2. Open the KEIL project C:\Users\{username}\ST\BlueNRG-LP_LPS DK x.x.x\Projects\Periph_Examples\LL\TIM\TIM_PWM_Train\MDK-ARM\{STEVAL-IDB011V1|STEVAL-IDB012V1}\TIM_PWM_Train.uvprojx
  3. Select desired configuration to build
  4. Select Project->Rebuild all target files. This will recompile and link the entire application
  5. To download the binary image, please connect an USB cable in your board (CMSIS-DAP upgrade).
  6. Select Project->Download to download the related binary image.
  7. Alternatively, open the BlueNRG-LP Flasher utility and download the built binary image.

Usage

Configuration of a timer to generate a fixed number of pulses in Output Compare mode. This example is based on the BLUENRG_LP TIM LL API. The peripheral initialization uses LL unitary service functions for optimization purposes (performance and size).

TIM1CLK is fixed to 64 MHz, the TIM1 Prescaler is set to have TIM1 counter clock = 10 Hz. The TIM1 auto-reload is set to generate PWM signal at 5 Hz. The channel 1 duty cycle is set to 25%. The update event (UEV) is generated after that the upcounting is repeated for the number of times programmed in the repetition counter register (TIMx_RCR + 1). The waveform generated is a train of 3 (TIMx_RCR + 1).

The PWM waveform can be displayed using an oscilloscope.

BlueNRG_LP-EVB Set-up Connect the following pins to an oscilloscope to monitor the different waveforms:

  • TIMx_CH4 Connect USART1 TX/RX to respectively RX and TX pins of PC UART (could be done through a USB to UART adapter) :
  • Connect BlueNRG_LP board USART1 TX pin to PC COM port RX signal
  • Connect BlueNRG_LP board USART1 RX pin to PC COM port TX signal
  • Connect BlueNRG_LP board GND to PC COM port GND signal

Launch serial communication SW on PC (as HyperTerminal or TeraTerm) with proper configuration :

  • 115200 bauds
  • 8 bits data
  • 1 start bit
  • 1 stop bit
  • no parity
  • no HW flow control
