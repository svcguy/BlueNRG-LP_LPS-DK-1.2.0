Detailed Description

How to configure USART peripheral to send characters synchronously to/from the LL SPI_IT_Master_Init example in DMA mode.

KEIL project

To use the project with KEIL uVision 5 for ARM, please follow the instructions below:

  1. Open the KEIL uVision 5 for ARM and select Project->Open Project menu.
  2. Open the KEIL project C:\Users\{username}\ST\BlueNRG-LP_LPS DK x.x.x\Projects\Periph_Examples\LL\USART\USART_SPI_Slave\MDK-ARM\{STEVAL-IDB011V1|STEVAL-IDB012V1}\USART_SPI_Slave.uvprojx
  3. Select desired configuration to build
  4. Select Project->Rebuild all target files. This will recompile and link the entire application
  5. To download the binary image, please connect an USB cable in your board (CMSIS-DAP upgrade).
  6. Select Project->Download to download the related binary image.
  7. Alternatively, open the BlueNRG-LP Flasher utility and download the built binary image.

Usage

This example shows how to configure GPIO and USART peripheral to send/receive characters synchronously to/from the LL SPI SPI_IT_Master_Init in DMA mode. This example is based on BLUENRG_LP USART LL API. Peripheral initialization is done using LL unitary services functions for optimization purpose (performance and size).

Example execution: After startup from reset and system configuration, the master LED is blinking quickly and wait User push-button (PUSH1) action. USART and DMA are configured. On press on push button , TX DMA transfer to SPI Slave application is initailize. RX DMA transfer is also ready to retrieve characters from MISO pin.

In case of both transfers successfully completed, LEDx is turned on. In case of errors, LEDx is blinking (1sec period).

In order to make the program work, you must do the following:

  • Launch serial communication SW on PC
  • Flash the project in the Board
  • Press the RESET button

To test this SPI Slave application was been used the LL/SPI/SPI_IT_Master_Init with the SPI configured as SPI1 and the data size set to 8bit. It is mandatory add to the slave application the follows define in the main.h file: #define CONFIG_DATASIZE_8BIT 1 #define USE_SPI1_PINS 1

| | STEVAL_IDB012V1 | | MASTER | SLAVE | | SPI Pin/Signal | USART Pin/Signal| | SCK | PB1 / CK | | MISO | PA1 / TX | | MOSI | PB0 / RX | | NSS | PB7 / NSS |

Note: NSS and nCTS share the same pin. LEDx is turned on in case of success. The aRxBuffer contains the received string.
