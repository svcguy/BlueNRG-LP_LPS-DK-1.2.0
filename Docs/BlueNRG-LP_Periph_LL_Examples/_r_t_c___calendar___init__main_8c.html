Detailed Description

Configuration of the LL API to set the RTC calendar.

KEIL project

To use the project with KEIL uVision 5 for ARM, please follow the instructions below:

  1. Open the KEIL uVision 5 for ARM and select Project->Open Project menu.
  2. Open the KEIL project C:\Users\{username}\ST\BlueNRG-LP_LPS DK x.x.x\Projects\Periph_Examples\LL\RTC\RTC_Calendar_Init\MDK-ARM\{STEVAL-IDB011V1|STEVAL-IDB012V1}\RTC_Calendar_Init.uvprojx
  3. Select desired configuration to build
  4. Select Project->Rebuild all target files. This will recompile and link the entire application
  5. To download the binary image, please connect an USB cable in your board (CMSIS-DAP upgrade).
  6. Select Project->Download to download the related binary image.
  7. Alternatively, open the BlueNRG-LP Flasher utility and download the built binary image.

  1. Open the Embedded Workbench for ARM and select File->Open->Workspace menu.
  2. Open the IAR project C:\Users\{username}\ST\BlueNRG-LP_LPS DK x.x.x\Projects\Periph_Examples\LL\RTC\RTC_Calendar_Init\EWARM\{STEVAL-IDB011V1|STEVAL-IDB012V1}\RTC_Calendar_Init.eww
  3. Select desired configuration to build
  4. Select Project->Rebuild All. This will recompile and link the entire application
  5. To download the binary image, please connect an USB cable in your board (CMSIS-DAP upgrade).
  6. Select Project->Download and Debug to download the related binary image.
  7. Alternatively, open the Flasher utility and download the built binary image.

  2. Select File, Import, Existing Projects into Workspace C:\Users\{username}\ST\BlueNRG-LP_LPS DK x.x.x\Projects\Periph_Examples\LL\RTC\RTC_Calendar_Init\WiSE-Studio\{STEVAL-IDB011V1|STEVAL-IDB012V1}
  3. Select desired configuration to build

  6. Select Run->Run/Debug to download the related binary image.

Usage

Configuration of the LL API to set the RTC calendar. The peripheral initialization uses LL unitary service functions for optimization purposes (performance and size).

The RTC peripheral configuration is ensured by the Configure_RTC() function (configure of the needed RTC resources according to the used hardware CLOCK, PWR, RTC clock source and BackUp). You may update this function to change RTC configuration.

Configure_RTC_Calendar function is then called to initialize the time and the date.

A key value RTC_BKP_DATE_TIME_UPDTATED is written in backup data register 1 to indicate if the RTC is already configured. To debug the intialize procedure when the backup data register 1 is been already written, is mandatory change the value of the #define RTC_BKP_DATE_TIME_UPDTATED.

When a reset (except power on reset) occurs the BKP domain is not reset and the RTC configuration is not lost.

When power on reset occurs and there are no battery connected to the VBAT pin: the BKP domain is reset and the RTC configuration is lost.

The current time and date are updated and displayed on the USART.

This example is applicable only for BlueNRG_LP cut 2.0 or above.

In order to make the program work, you must do the following:

  • Launch serial communication SW on PC
  • Flash the project in the Board
  • Press the RESET button

BlueNRG_LP-EVB Set-up Connect USART1 TX/RX to respectively RX and TX pins of PC UART (could be done through a USB to UART adapter) :

  • Connect BlueNRG_LP board USART1 TX pin to PC COM port RX signal
  • Connect BlueNRG_LP board USART1 RX pin to PC COM port TX signal
  • Connect BlueNRG_LP board GND to PC COM port GND signal

Launch serial communication SW on PC (as HyperTerminal or TeraTerm) with proper configuration :

  • 115200 bauds
  • 8 bits data
  • 1 start bit
  • 1 stop bit
  • no parity
  • no HW flow control
