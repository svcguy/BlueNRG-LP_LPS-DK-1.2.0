uint32_t FLASH_PageProtectionTypeDef::pageProt_0

For cut 1.0 indicate the group to configure.

  • This parameter can be a value of FLASH write/page erase protection for group For cut 2.0 it is a bitfield built as follows:
  • bit[6:0] = SIZE: number of 2 kB pages to protect including the starting page pageProt_0 param (provided in the offset bits)
  • bit[7] = reserved
  • bit[14:8] = OFFSET: page number to start the write protection (value between 0 and 0x7F)
  • bit[15] = reserved
  • bit[22:16] = SIZE: same meaning of bit[6:0]
  • bit[23] = reserved
  • bit[30:24] = OFFSET: same meaning of bit[22:16]
  • bit[31] = reserved
