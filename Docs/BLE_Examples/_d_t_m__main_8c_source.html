259 /*
260  ******************************************************************************
261  ******************************************************************************
262  *
263  * The DTM project includes the following configurations:
264  *
265  * Configuration | Description
266  * UART/SPI | The DTM with interface UART/SPI
267  * UART/SPI_WITH_UPDATER | The DTM including the DTM_Updater in the first page
268  * | of the Flash memory. This image is necessary to use
269  * | the IFR Tool (GUI) and the DTM Firmware Updater (GUI).
270  * UART_FOR_UPDATER | The DTM image with the first page empty (offset 0x200).
271  * | This image needs the DTM_Updater to run and it is
272  * | used for the DTM Firmware Updater (GUI).
273  *------------------------------------------------------------------------------
274  *
275  * The configuration DTM - UART_WITH_UPDATER has the following memory layout:
276  *
277  * ------------- 0x1007FFFF
278  *
279  * DTM
280  *
281  * ------------- 0x10042000
282  * DTM Updater
283  * ------------- 0x10040000
284  *
285  * The DTM Updater allows to access the memory flash through ACI_HAL commands.
286  * The DTM Updater can be activated in the following way:
287  * 1) Activation by using ACI_HAL_UPDATER_START
288  * 2) Activation by using PA15 pin (high level at start up).
289  * Note: if the PA15 pin is used and is high at start up, this will cause the
290  * DTM Updater starts. So, to avoid this, the support of the DTM Updater
291  * can be removed.
292  *
293  * Any change of relevant pins must be reported
294  * also in DTM Updater firmware.
295  *
296  ****************************************************************************
297  ****************************************************************************
298 */
