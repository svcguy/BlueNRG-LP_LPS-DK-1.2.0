Bluetooth LE stack v3.x: GAP APIs/events
Scanning/discovery procedures steps and APIs
  • Scanning/discovery procedures are handled in 3 steps:
    1. Configure Scanning parameters (API: aci_gap_set_scan_configuration())
    2. Start Discovery procedure (API: aci_gap_start_procedure(())
  • Step 1 allows to define the scanning/discovery specific parameters (scanning interval min and max, ...). User can change these parameters when he needs by calling the related API.
  • Step 2 allows to start a specific discovery procedure based on user application needs. User must configure scanning parameters at least one time before using this API.
  • NOTE:
    • These APIs also allows to configure and set extendend scanning procedures.
