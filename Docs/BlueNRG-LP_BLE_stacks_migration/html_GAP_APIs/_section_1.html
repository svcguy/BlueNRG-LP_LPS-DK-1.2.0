Bluetooth LE stack v3.x: GAP APIs/events
Advertising modes steps and APIs
  • Advertising mode is handled in 3 steps:
    1. Configure Advertising parameters (API: aci_gap_set_advertising_configuration())
    2. Configure Advertising data (API: aci_gap_set_advertising_data())
    3. Enable/disable Advertising (API: aci_gap_set_advertising_enable())
  • Step 1 allows to define the advertising specific parameters (advertising interval min and max, ...). User can change these parameters whenever he needs by calling the related API.
  • Step 2 allows to define the advertising data according to the Bluetooth LE specification type and format (ADType flags, Complete or Short Name). User must follow Bluetooth LE Specification rules in order to correctly format advertising data and he can change these parameters whenever he needs by calling the related API.
  • Step 3 allows to enable and disable advertising based on user application needs. User must configure advertsing parameters and related data at least one time before using this API.
  • NOTEs:
    • These APIs also allows to configure and set extended advertising data.
